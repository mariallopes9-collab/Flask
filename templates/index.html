01 — O que é Python?
Linguagem de Programação

Conhecendo o Python

Python é uma linguagem de programação de alto nível, interpretada e de propósito geral, criada por Guido van Rossum e lançada em 1991. Seu design prioriza a legibilidade do código, usando identação (espaços) para definir blocos, o que torna o código quase tão legível quanto texto em inglês.

É uma linguagem multiparadigma: suporta programação imperativa, funcional e orientada a objetos — e possui tipagem dinâmica e gerenciamento automático de memória.

  • Web — servidores, APIs REST, microsserviços (Flask, Django, FastAPI)
  • IA & Machine Learning — TensorFlow, PyTorch, scikit-learn
  • Ciência de Dados — análise e visualização com Pandas e Matplotlib
  • Automação — scripts, bots, automação de tarefas repetitivas
  • Segurança — testes de penetração e ferramentas de cibersegurança
  • Games — prototipagem rápida de jogos com Pygame
Carreira & Mercado

Vantagens Profissionais

Python é a linguagem mais popular do mundo segundo o índice TIOBE e o Stack Overflow Survey. Dominar Python abre portas em praticamente todas as áreas de tecnologia.

  • Alta demanda no mercado — empresas de todos os setores buscam desenvolvedores Python
  • Salários competitivos — especialmente para Data Science, IA e Back End
  • Produtividade — sintaxe clara = código mais rápido de escrever e manter
  • Ecossistema enorme — mais de 400 mil pacotes no PyPI para praticamente tudo
  • Fácil aprendizado — ideal para iniciantes, mas poderoso o suficiente para especialistas
  • Comunidade global — vasta documentação, tutoriais e suporte online
02 — POO
Paradigma de Programação

Programação Orientada a Objetos

A Programação Orientada a Objetos (POO) é um paradigma que organiza o software em torno de objetos — entidades que combinam dados (atributos) e comportamentos (métodos). Em vez de escrever uma série de funções soltas, modelamos o mundo real em "moldes" chamados classes.

Uma classe é como uma planta baixa; um objeto é a casa construída a partir dela. Podemos criar quantos objetos quisermos a partir de uma mesma classe.

Encapsulamento

Oculta os detalhes internos e expõe apenas o necessário, protegendo os dados.

Herança

Uma classe filha herda atributos e métodos da classe pai, evitando repetição de código.

Polimorfismo

O mesmo método pode se comportar de formas diferentes dependendo do objeto que o usa.

Abstração

Simplifica a complexidade, focando apenas nos aspectos relevantes do problema.

Por que usar POO?

  • Reuso de código — escreva uma vez, use em muitos lugares via herança
  • Manutenção fácil — alterações isoladas em uma classe não quebram o restante
  • Organização — código estruturado em módulos lógicos e intuitivos
  • Trabalho em equipe — cada desenvolvedor trabalha em sua classe sem conflito
